Chapter 4: Data Communication and Computer Networks

Comprehensive Notes, Questions, and Practice for Sindh Textbook Board

Chapter Notes

This chapter covers the fundamentals of data communication, transmission media, network types, and network topologies.

What is Data Communication?

Data communication refers to the exchange of data between two devices via some form of transmission medium.

Components of Data Communication

  • Message: Information to be communicated.
  • Sender: Device that sends the data message.
  • Receiver: Device that receives the message.
  • Transmission Medium: Physical path for message transmission.
  • Protocol: Set of rules governing data communication.

Data Transmission Modes

  • Simplex: One-way communication (e.g., keyboard to computer).
  • Half-Duplex: Two-way communication, but not simultaneously (e.g., walkie-talkie).
  • Full-Duplex: Two-way simultaneous communication (e.g., telephone).

Transmission Media

  • Guided Media: Wired transmission (twisted pair, coaxial cable, fiber optics).
  • Unguided Media: Wireless transmission (radio waves, microwaves, infrared).

Computer Networks

A computer network is a collection of interconnected computers and devices to share resources and information.

Types of Networks

  • LAN (Local Area Network): Connects devices in a limited area (e.g., school, office).
  • MAN (Metropolitan Area Network): Covers a larger area like a city.
  • WAN (Wide Area Network): Spans large geographical areas (e.g., internet).

Network Topologies

  • Bus Topology: All devices connected to a single backbone cable.
  • Star Topology: All devices connected to a central hub/switch.
  • Ring Topology: Devices connected in a circular fashion.
  • Mesh Topology: Every device connected to every other device.

Network Devices

  • Modem: Modulates and demodulates signals for transmission.
  • Router: Connects different networks and directs data packets.
  • Switch: Connects devices within a network and forwards data.
  • Hub: Basic networking device that connects multiple devices.

Network Protocols

TCP/IP, HTTP, FTP, SMTP - rules that govern data communication over networks.

Long Questions

  1. Explain the components of data communication with examples.
    Answer: The five components are: Message (data to be communicated), Sender (device that sends data), Receiver (device that receives data), Transmission Medium (physical path), and Protocol (rules for communication). Example: Sending an email where computer is sender, message is email content, internet is medium, recipient's computer is receiver, and SMTP is protocol.
  2. Describe different types of computer networks.
    Answer: LAN (Local Area Network) connects devices in a limited area like a school or office. MAN (Metropolitan Area Network) covers a larger area like a city. WAN (Wide Area Network) spans large geographical areas like countries. PAN (Personal Area Network) connects devices in personal workspace.
  3. Compare and contrast different network topologies.
    Answer: Bus topology uses a single backbone cable but if it fails, entire network fails. Star topology connects all devices to a central hub - easy to install but hub failure affects all devices. Ring topology connects devices in a circle - data travels in one direction but failure of one device can break the ring. Mesh topology connects every device to every other device - highly reliable but expensive and complex.
  4. Explain different data transmission modes with examples.
    Answer: Simplex mode allows one-way communication only (e.g., keyboard to computer). Half-duplex allows two-way communication but not simultaneously (e.g., walkie-talkie). Full-duplex allows simultaneous two-way communication (e.g., telephone conversation).
  5. Discuss the importance of network protocols with examples.
    Answer: Protocols ensure standardized communication between devices. TCP/IP enables internet communication. HTTP allows web browsing. FTP facilitates file transfers. SMTP handles email transmission. Without protocols, different devices couldn't communicate effectively.

Numerical Problems

1. Calculate the time required to transfer a 2 MB file over a network with a bandwidth of 10 Mbps.

Solution: 2 MB = 16 Mb (since 1 byte = 8 bits). Time = Data Size / Bandwidth = 16 Mb / 10 Mbps = 1.6 seconds.

2. If a network has 12 computers in a mesh topology, how many cables are needed?

Solution: In mesh topology, number of cables = n(n-1)/2 where n is number of devices. So, 12(12-1)/2 = 12*11/2 = 66 cables.

3. A file of 5 MB is being downloaded at 2 MBps. How long will it take?

Solution: Time = File Size / Download Speed = 5 MB / 2 MBps = 2.5 seconds.

4. Calculate the number of connections in a star topology with 8 computers.

Solution: In star topology, each computer connects to a central hub. So, number of connections = number of computers = 8.

5. If a signal takes 0.002 seconds to travel through a cable, what is the latency?

Solution: Latency is the time delay in transmission. Here, latency = 0.002 seconds or 2 milliseconds.
